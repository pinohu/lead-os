# Provider Dashboard

## Purpose

Give paying territory owners a clear, operational view of the leads they receive, their territory status, and basic conversion metrics.

## Core Dashboard Sections

- territory status
- recent leads
- delivery status
- conversion tracking
- billing status

## Data Sources

- ownership state
- subscription state
- intake records
- delivery events
- CRM sync status

## SuiteDash Role

SuiteDash acts as the provider-facing CRM and portal destination.
The runtime dashboard should mirror the critical operating data and provide the payloads needed for SuiteDash automation.

## Core Rule

Only active exclusive owners should have access to a node dashboard for that territory.

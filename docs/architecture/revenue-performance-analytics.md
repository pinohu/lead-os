# Revenue and Performance Analytics Dashboard

## Purpose

Provide operators with business visibility into territory performance, provider ROI signals, revenue health, and conversion throughput.

## Core Panels

- revenue summary
- subscription summary
- territory ranking
- provider lead volume
- delivery success summary
- churn risk summary

## Core Rule

Operators should be able to see which nodes, providers, and subscriptions create value and which ones need intervention.

## Data Sources

- leads
- subscriptions
- node_ownership
- delivery_jobs
- dead_letter_jobs

## Initial Scope

- total leads
- active subscriptions
- estimated MRR
- leads by node
- leads by owner
- delivery success/failure counts

## Future Scope

- booking rates
n- close rates
- revenue attribution by territory
- churn prediction

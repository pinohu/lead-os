# Operator Dashboard

## Purpose

Provide one command center for running the marketplace: system health, revenue, pricing approvals, churn risk, and territory performance.

## Core Panels

- system health
- revenue summary
- pricing approvals queue
- territory ranking
- churn risk watchlist
- dead-letter and delivery failures

## Core Rule

Operators should be able to understand the business state and take action from one place.

## Initial Scope

- summary cards
- recent pricing recommendations
- recent failures
- top territories
- provider ROI snapshot
- churn risk list

## Future Scope

- approval actions in UI
- replay dead-letter jobs
- bulk territory repricing
- operator notes and acknowledgements

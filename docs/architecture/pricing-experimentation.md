# Pricing Experimentation Engine

## Purpose

Test pricing strategies safely across territories and learn which prices maximize revenue, conversion, and retention.

## Core Flow

1. define experiment
2. assign eligible territories to variants
3. apply bounded pricing variants
4. measure outcomes
5. evaluate winner
6. promote winning policy

## Core Rules

- experiments must be bounded by safety thresholds
- grandfathered accounts are excluded by default
- experiments should run on eligible low-risk territories only
- experiment outcomes must be measured against revenue, conversion, and churn indicators

## Initial Variant Types

- control: current price
- variant_a: moderate increase
- variant_b: stronger increase
- rescue: discount or hold strategy for low-performing territories

## Success Metrics

- revenue per territory
- revenue per lead
- close rate
- churn rate
- provider feedback score
